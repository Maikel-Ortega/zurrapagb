Behavior

System: YAMAHA YMU759 ››

Here are some descriptions of the current system's behavior:

Emulator issues on Windows Vista:
Because of the nature of the YMU759 Emulator, it is possible to suffer some problems with some Windows Vista sound drivers. I recommend using Windows 7 or XP if you want to make some tunes using the YMU759.

Wav Exporting:
Although it is possible to export WAV files on the other supported systems, is not possible to export a WAV file of the song using the YMU759. The way in which the emulator works does not permit read the output generated. WAV exporting using the YMU759 can be possible on future versions, but I need to hack the emulator dll or switch to other emulator, and there are other priorities right now.
However, you can always record the output of your soundcard while DefleMask is playing, or record a video showing your tracking skills.

Instrument Column:
The YMU759 is not capable to switch instruments on the fly, so the instrument column will only accept the first value of the first row to set the channel instrument. All instrument changes would be translated to the first row of the first pattern.

Volume column:

You can designate a value of 0~7F, but there are ranges in which the volume does not change.

For example the volume does not change for 0~3.

Effects Behavior:

Effect 4 - Vibrato/Modulation:

You can designate a value of 0~7F, but DefleMask, internally, recognizes only five levels:


0 Turn vibrato off for all operators.

1~1F Vibrato will be as designated by the instrument DVB Value.

1F~3F Add +1 to the Vibrato DVB value of the instrument.

40~5F Add +2 to the Vibrato DVB value of the instrument.

60~7F Add +3 to the Vibrato DVB value of the instrument.

If adding to the DVB value would cause DVB to exceed +3, the result will be +3.


Effect 8 - Panning:

You can designate a value of 0~7F.
0 means full left, 7F means full right.

You can obtain specific panning values using intermediate values, such as 5A.
